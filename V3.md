# Master Implementation Plan: Nvidia AI Studio v3.0
*The autonomous multi-agent operating system exclusively for macOS.*

This document outlines the architectural changes required to transition Nvidia AI Studio to version 3.0, incorporating a 24/7 God-Mode Dashboard alongside the 5 visionary pillars of productivity.

---

## Pillar 1: 🧠 Swarm Mode & The "War Room" (Spatial UX)

We are breaking away from transient background processes and moving to a true background "Swarm" of autonomous agents. The UI paradigm shifts from a "messaging app" to an **Operations Office (Spatial UX)**.

> [!TIP]
> **Database Architecture Decision**: `SwiftData` has been officially selected as the backend layer over JSON files. SwiftData will natively manage the continuous queue of tasks without blocking the UI thread or risking file corruption.

### 🏢 The War Room Concepts (Spatial UX)

Instead of a generic 1-on-1 chat, the interface mimics a physical working environment.

1. **The Operations Floor (Dashboard)**:
   - A spatial layout featuring interactive **Character Avatars** (not generic squares). Each avatar visually represents a unique Swarm Agent (e.g., Code Reviewer, UI Designer, Backend Dev).
   - You can see exactly what each avatar is doing (their live PTY terminal acts as their "computer monitor").
2. **The Debate Room (Conference Room)**:
   - A specialized multi-agent environment.
   - You drop a complex task on the table (e.g., "Design a resilient database architecture") and place *Architect* and *Security Expert* agents in the room.
   - They read each other's messages, debate the best approach autonomously, and arrive at a consensus. You supervise as the CEO and can interject.
3. **The Archives / Inbox (Filing Cabinet)**:
   - Output from agents is treated as physical "Deliverables".
   - When an agent finishes writing a script, scraping a website, or fetching a memory, they do not dump 5 pages of text into a chat window.
   - They "drop" a sealed Object (a File, a Report, a Schema) into your **Inbox**. Clicking it opens a clean viewer.

### Architecture Changes (Data Layer)
- **Data Layer (SwiftData)**: 
  - `Models/AgentPersona.swift` (`@Model`): Defines the worker.
  - `Models/SwarmTask.swift` (`@Model`): The queued jobs.
  - `Models/SwarmDeliverable.swift` (`@Model`): [NEW] The "physical" files or reports agents place in the Inbox.
- **Swarm Orchestrator (`ViewModels/SwarmOrchestrator.swift`)**: The background `@Observable` multi-agent engine.

## Pillar 2: 🤖 "Auto-Pilot" Terminal (Agent-in-the-Loop)
Connecting the agent's brain to the MacOS file system directly.

### Architecture Changes
- **PTY Binding**: Dedicated `PTYProcess` for each agent.
- **Raw Input (Tab Completion & History)**: Invisible `NSView` key-event responder for native byte injection.
- **Telemetry UI**: Clicking an agent's "Desk" zooms into their live PTY output.

---

## Pillar 3: 🌍 Live Canvas (App Previewer)
When agents build React/Vite web apps, you should see them function instantly.

### Architecture Changes
- **WebView Integration (`Views/RightPanel/LiveCanvasView.swift`)**: Apple's `WKWebView` integration.
- **Auto-Host Resolution**: Regex detection on PTY for `localhost` auto-navigation.
- **Vision Loop**: Autonomous JS error detection via screenshot injection.

---

## Pillar 4: ⚡ Global Spotlight (macOS Integration)
A floating prompt accessible from anywhere on the Mac.

### Architecture Changes
- **Global Hotkey Registration**: `Cmd + Space` hook via `NSEvent`.
- **Context Injection API**: Pull active MacOS window title via Accessibility APIs and inject into Swarm.

---

## Pillar 5: 🔌 MCP (Model Context Protocol) 

### Architecture Changes
- **Swarm Tool Inheritance**: Orchestrator dynamically fetches tools from the existing `SkillRegistry` populated by active MCP servers.
- **Headless MCP Execution**: Stdio streams handle payloads cleanly.

---

## Pillar 6: 🎨 UI Consistency & New Themes

### Architecture Changes
- **Settings View Theming**: Unified hierarchy injection for background coherence.
- **New App Themes**: *Lights Out (OLED Dark)* and *Nord / Arctic (Light)*.

## Action Plan Outline

When development for v3.0 begins on the DEV branch, execution will follow these distinct phases:

1. **Phase 1: Foundation & Persistence** (SwiftData + SwarmOrchestrator). The invisible layer that makes 24/7 autonomy possible. *(Currently executing via task.md)*
2. **Phase 2: The Command Center UI** (SwarmDashboard + The Archives). Shifting to Spatial UX.
3. **Phase 3: Conference & Debate Rooms** (Multi-Agent Swarm instances).
4. **Phase 4: Agent Senses** (PTY Auto-Pilot Binding + Live Canvas).
5. **Phase 5: macOS Deep Integration** (Global Spotlight + MCP hooks).
6. **Phase 6: UI Polish & Theming** (Settings consistency + New Themes).
